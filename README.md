# ms2TreeHelper

`ms2TreeHelper` — небольшое дополнение для **MODX Revolution** и **miniShop2**, которое добавляет массовые действия в дерево категорий в окне создания и редактирования опций miniShop2.

Компонент решает практическую проблему: когда опцию нужно привязать сразу к большому числу категорий, в стандартном интерфейсе приходится отмечать чекбоксы по одному. `ms2TreeHelper` добавляет удобные кнопки прямо в toolbar дерева категорий.

## Что делает компонент

В окне создания/редактирования опции miniShop2 добавляются три кнопки:

* **Добавить на все категории**
* **Добавить ко всем дочерним категориям выбранной категории**
* **Снять со всех категорий**

Кнопки встроены в интерфейс менеджера и работают без правок core MODX и core miniShop2.

## Преимущества

* не требует изменений в `core/components/minishop2`
* не ломает стандартную логику сохранения miniShop2
* подключается как отдельный extra
* подходит для проектов с большим деревом категорий
* ускоряет работу контент-менеджера и администратора
* может использоваться как готовое решение для клиентских проектов

## Как работает

Компонент подключает собственный JavaScript в manager MODX и расширяет интерфейс окна опций miniShop2. Вся логика реализована поверх штатного дерева категорий, поэтому компонент не вмешивается в процессоры miniShop2 и не изменяет структуру базы данных.

## Совместимость

Проверялось на:

* **MODX Revolution 2.8.x**
* **miniShop2 3.x**

При необходимости код можно адаптировать и под другие версии, если структура окна в manager отличается.

## Установка

1. Установить пакет `ms2treehelper` через **Extras → Installer**.
2. Очистить кэш MODX.
3. Обновить страницу manager.
4. Открыть **miniShop2 → Настройки → Опции**.
5. При создании или редактировании опции использовать новые кнопки над деревом категорий.

## Безопасность изменений

Компонент разработан так, чтобы не изменять core-файлы MODX и miniShop2. Это упрощает обновления системы и снижает риск конфликтов при сопровождении проекта.

## Для кого это решение

`ms2TreeHelper` полезен, если:

* в магазине много категорий
* опции часто нужно назначать сразу на группы разделов
* ручное проставление чекбоксов занимает слишком много времени
* нужен аккуратный extra без вмешательства в ядро проекта
